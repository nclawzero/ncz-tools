# zterm v0.3 UX roadmap — Paradox 4.5 / dBASE V aesthetic

*Locked 2026-04-22 after an extended design session (10+ DOS-era
reference screenshots) and a GRAEAE multi-LLM consensus consult.*

## Aesthetic authority

**Paradox 4.5 (1993) + dBASE V for DOS (1994)** — the last two
Borland DOS releases, Turbo Vision 2.x at its absolute peak before
the Win16 transition. All visual calls defer to this reference.
Secondary references for cross-checking: Midnight Commander, PFS
Professional Write/File, dBASE III Plus, FoxPro, Procomm Plus 2.x.

When a visual decision isn't obvious from the Paradox 4.5/dBASE V
reference, prefer "what would Turbo Vision have done" over novel
invention.

## Substrate decisions

### `turbo-vision-4-rust` over cursive / ratatui

`turbo-vision-4-rust` is a pure-Rust implementation of Borland's
classic Turbo Vision UI framework — the actual framework that
powered Turbo C++, Turbo Pascal, Paradox 4.5, dBASE V, Quattro
Pro, Sidekick, and every other peak-Borland DOS app. Not an
approximation, not a theme on top of a generic TUI toolkit —
the thing itself, reborn in Rust with a crossterm backend.

- crates.io: `turbo-vision` v1.1.0
- Repo: https://github.com/aovestdipaperino/turbo-vision-4-rust
- License: MIT (compatible with zterm's Apache-2.0)
- Crossterm backend — Linux / macOS / Windows
- Status: version 1.0.0 claims "PRODUCTION READY"
- ~30+ examples covering dialogs, editor, file browser,
  function-key menu, cascading menus, palette themes, tree
  view, validator widgets, etc.
- Actively maintained (last commit within weeks)

The widget vocabulary zterm needs — `TDialog`, `TWindow`,
`TMenuBar`, `TStatusLine`, `TListBox`, `TInputLine`,
`TPalette`, drop-shadows, CP437 box chars, cascading
submenus with accelerator-letter jumps — is the library's
native surface. No imitation cost, no theme-gymnastics to
match Paradox 4.5 because the framework was literally the
one Paradox 4.5 used.

Rejected alternatives:

- **cursive** — close but not the actual thing. Would require
  manual theme tuning to match Turbo Vision exactly. Chosen
  initially during design but superseded when
  turbo-vision-4-rust was surfaced.
- **raw ratatui** — too low-level, would require building
  every Turbo Vision widget from scratch.
- **native GUI via wgpu/winit** — reinventing what
  Ghostty/Kitty/Warp/WezTerm already solved. zterm is not a
  terminal emulator.

### Alt-screen mode, not native GUI window

Early in the design session we considered spawning our own native
window (via `wgpu`/`winit` or embedding a terminal-emulator
backend like `alacritty_terminal`). Rejected: reinventing what
Ghostty/Kitty/Warp/WezTerm already solved, and zterm is not a
terminal emulator — it's an agent REPL.

Instead, zterm runs in alt-screen mode (`\e[?1049h`), which every
modern terminal emulator supports (vim/htop/btop/mc all do this).
No dependency on any specific host. Scrollback-vs-TUI conflicts
disappear by definition: alt-screen has no scrollback.

### Tokio ↔ turbo-vision bridge

- Turbo Vision event loop runs on the main thread via
  `TApplication::run()`
- `tokio` runtime runs on a dedicated thread with a multi-thread
  scheduler for the WebSocket/HTTP I/O to agent daemons
- Communication via `tokio::sync::mpsc` channels (request side)
  and a custom `TEvent::Message(payload)` posted via
  `TApplication::put_event()` (UI update side — turbo-vision's
  equivalent of cursive's `cb_sink`)
- Same bridge shape as any async-bridged retained-mode TUI
  (ncspot-on-cursive is the industry reference)

## Palette — Paradox 4.5 calibration

| Element | Color | Use |
|---|---|---|
| Main background | Mid-navy | Fills the whole screen behind panels |
| Panel / data-view body | Cyan | Dialog/data interiors |
| Panel borders | Yellow (bright) | Frame chars around every Panel and Dialog |
| Dialog title | White or cyan on black | Two-tone title block on top of body |
| Selection bar | Bright yellow | Highlight row in any `SelectView` |
| Selection text | Dark navy | High contrast against yellow bar |
| Accelerator letter | Cyan (bright) | Single-letter keyboard shortcut per menu item |
| Drop shadow | Pure black | 1×2 or 1×3 cell offset below-right of modals |
| Dangerous action accent | Coral/salmon | Exit, Delete, Destroy — used sparingly |

## Theme presets (runtime-switchable via `/theme <name>`)

| Preset | Target | Notes |
|---|---|---|
| `pfs` (default) | Paradox 4.5 color | Calibrated per the palette table above |
| `mono` | dBASE III monochrome | Reverse-video only, no color, black/white contrast |
| `amber` | IBM 3270 / amber CRT | `#FFB000` amber on black, single tone |
| `green` | VT100 / P1 phosphor CRT | `#33FF33` green on black, single tone |
| `borland` | Cyan Borland IDE look | Cyan main bg (not navy), otherwise similar to `pfs` |

Writable at `~/.zterm/theme.toml`. Custom per-element color
customization lands in **E-8** (Borland-style two-column
foreground/background picker).

## Functional primitives

Every visual surface serves a function first and carries retro
styling as wrapping. "Retro joy, not retro regret."

- **F-key menu bar** across the top. Each cell dispatches a real
  action (help, save, workspace, models, memory). Active menu
  highlighted in coral/salmon when its dropdown is open.
- **Slash-command popup** via `/` key. Centered modal with
  cascading submenus that open to the right (Paradox-style).
  Each item has a cyan accelerator letter; typing that letter
  jumps to the item. Arrow keys + Enter to select.
- **Status bar** just above the input line: `workspace · model ·
  ctx {used}/{total} ({pct}%) · {mm:ss elapsed}`. Token counts
  pulled live from `TurnResult.usage` after each turn.
- **Context hint line** at the very bottom. Updates with the
  current selection or active widget. Always shows actionable
  key bindings: `↩ Select · Esc Cancel · ↑↓ Navigate · F1 Help`.
- **Input line** above the hint strip. Multi-level undo stack
  (Ctrl-Z). Insert/overwrite cursor modes (block vs underscore).
  Standard modern copy/paste bindings (Ctrl-C/V/X) left alone.
- **Modal pickers** for `/workspace switch`, `/models list`, and
  theme editor — always framed, drop-shadowed, with context
  hint strip at the bottom of each modal.

## Slice plan

### Chunk E — Experience (the real UX rewrite)

- **E-1** turbo-vision scaffold: `TApplication` with
  `TMenuBar` across the top (File · Session · Workspace ·
  Model · Memory · Help), `TDesktop` main area, `TStatusLine`
  at the bottom with contextual key hints, a main `TWindow`
  hosting the chat pane + input line. Input echoes locally;
  no backend wiring yet. Theme = Turbo Vision default
  (cyan-on-blue). rustyline path kept behind `--legacy-repl`
  for fallback during transition.
- **E-2** tokio↔turbo-vision bridge. `tokio::sync::mpsc`
  channel carries `Request::SubmitTurn(text)` from the TUI
  thread to a tokio worker; streamed events return via
  `TApplication::put_event(TEvent::Message(...))` and append
  to the chat pane's `TMemo` / `TTextDevice`.
- **E-3** slash popup wired to real `CommandHandler`. Uses
  `TSubMenu` / `TMenuItem` with cascading submenus, hotkey
  (accelerator-letter) jumps, and the status line updating
  contextually per selection. **v0.3.1:** menu and slash-popup
  entries now dispatch real workspace, model, provider, MCP,
  memory, session, help, and local theme actions through
  `CommandHandler` or the existing TV theme path.
- **E-4** status bar real data: `TStatusLine` segments for
  workspace label, model from session, ctx tokens from
  `TurnResult.usage`, elapsed timer. **v0.3.1:** `TurnUsage`
  propagates from zeroclaw/openclaw stream events and renders
  `ctx used/total (%)` with a compact token-budget bar when
  the backend provides a context window.
- **E-5** modal pickers for `/workspace switch` (`TDialog`
  with `TListBox`) and `/models list` (nested `TDialog` +
  `TListBox` grouped by provider). **v0.3.1:** workspace and
  session picker paths are wired through `MenuBox`; full modal
  picker polish remains deferred.
- **E-6** theme presets via `TPalette` swaps: `pfs` (Paradox
  4.5 calibration), `mono` (dBASE III reverse-video), `amber`
  (CRT amber), `green` (phosphor green), `borland` (cyan
  Turbo Vision default). Runtime switch via `/theme <name>`
  with instant redraw. **Update:** also bound to a direct
  cycler key — `Ctrl-P` walks `themes::PRESETS` in order with
  a 1-second status-line toast confirming the active preset,
  no slash command needed. Selection persists to
  `~/.zterm/theme.toml` on every cycle so a restart picks it
  up. Implementation: `themes::next_preset` + `CMD_PALETTE_NEXT`
  in `tv_ui::handle_command`.
- **E-6b** multi-level undo stack in `TInputLine` buffer.
- **E-7** Animated spinner during in-flight turns +
  active-menu highlight on `TMenuBar`. **v0.3.1:** spinner
  timing is test-covered and the color themes use a coral
  active-menu slot.
- **E-8** Borland-style theme editor: two-column fg/bg color
  picker `TDialog` per UI element, writes to
  `~/.zterm/theme.toml`. (This matches the actual dBASE
  color-customization dialog we saw as reference.)

### Chunk F — Flavor (small joy-per-effort delighters)

- **F-1** LLM-generated connect-splash sequence on workspace
  activation. Fires a tiny turn at the active daemon's model:
  "generate a 4-6 line 1991 BBS modem connect sequence." Output
  typewriter-rendered at 30ms/char into the main pane before
  the welcome banner. Cached per-workspace per-session.
  **v0.3.1:** cache is `~/.zterm/cache/connect-splash/<workspace>.txt`
  with 24h mtime invalidation; boot and runtime workspace
  switches use the same generator/cache path.
- **F-2** `--wopr` mode. Canned fair-use WarGames LOGON:Joshua
  sequence (ends at "SHALL WE PLAY A GAME?"). Sourced from the
  `built1n/wargames` public transcription, attributed in
  `NOTICE`. Typewriter-rendered on first session boot when flag
  is set.
- **F-3** Welcome-back easter egg on Nth launch (e.g. 5, 10, 25)
  — randomized period-accurate quote ("Turbo Pascal says: Hello,
  world!" / "WordStar 7 loaded. Press ^KD to save."). Per-launch
  counter stored in `~/.zterm/state.toml`. **v0.3.1:** shipped
  for launches 5, 10, 25, 50, and subsequent multiples of 25.
- **F-4** Configurable beep-on-error. Off by default. Toggle via
  `/theme beep on|off`. Respect terminal bell. **v0.3.1:** stored
  as `beep_on_error` in `~/.zterm/state.toml` and emitted only on
  error frames.
- **F-5** Subtle phosphor fade-in on theme switch and on launch.
  ~200 ms opacity ramp via staged redraws. No spinning loader,
  no progress bar. Just a single "whoa it's alive" visual breath.

## GRAEAE-consensus design principles

These came back from a multi-LLM consult during the design
session (perplexity/sonar-pro, gemini-3.1-pro,
all models responded). Convergent enough that
we treat them as locked design constraints.

### DO

- Typewriter streaming at the daemon's natural token speed. Do
  not artificially delay tokens. Let the speed of the model be
  the "baud rate."
- Responsive layout: fill the terminal, but cap chat/content
  reading width to 80–100 chars. Forcing 80×25 on a modern
  monitor = regret. Use extra horizontal space for side panels.
- Unicode box-drawing (not strict CP437). Universally supported,
  visually indistinguishable, no capability-detection code.
- Blinking cursor: block for overwrite, underscore for insert.
  Toggle with `Insert` key per period convention.
- Drop-shadow on every modal overlay.
- F-key and Alt+letter retro bindings PRIMARY; modern fallbacks
  SECONDARY but fully supported.
- Context hint line always visible and always correct.

### AVOID

- Single-line undo. Multi-level undo is mandatory even though
  period DOS apps often shipped with single-line. GRAEAE flagged
  this as "deep regret" from Gemini — the joy-killer.
- Overriding Ctrl-C/V/X for non-copy/paste purposes. Leave those
  for modern muscle memory; bind retro conventions to Alt- and
  F-keys.
- Gratuitous loading bars or ASCII art everywhere. Kills retro
  credibility instantly (reads as kitsch).
- Theme incoherence — mixing eras or palettes mid-session.
- Slow startup. Forced-resize modal alerts. Any "this terminal
  is too small" interruption.

## Philosophical frame (Gemini's framing, adopted verbatim)

> **Turbo Vision UX with Rust performance.** The joy comes from
> the snappy, predictable, and tactile nature of those old
> interfaces, while the regret usually comes from their hardware
> limitations.

We keep what was *good* about late-80s DOS (keyboard mastery,
contextual affordances, immediate feedback, predictable layout)
and drop what was *bad* (limited undo, slow redraws, forced
80×25, beep-on-everything, opaque errors).

## Versioning

The Turbo Vision render path shipped under the **v0.3.x** track.
v0.3.1 closes the remaining targeted E/F slices while leaving
full E-5 picker polish, full E-8 theme editor expansion, WOPR
mode, and phosphor fade-in deferred.
